Anexo 1: Formato de Plan de Pruebas

(Basado en IEEE 829)
Semana 3 - Sesión 8
Proyecto de Equipo
IEEE 829

📋 Instrucciones para el Alumno

Cuándo presentarlo: Semana 3, Sesión 8 (El Plan de SQA)

Nota: Este formato es el principal entregable de la planificación de las actividades de Control de Calidad (QC) dentro de un Plan de SQA. Debe llenarse con base en el proyecto de equipo.

1
Identificador del Plan de Pruebas
Escribe un código único para identificar este plan, indicando nombre del proyecto y versión.
Ejemplo: PP-PROYECTO_X-V1.0
Identificador
2 Introducción y Referencias
Breve descripción del proyecto y alcance del plan. Lista de documentos de referencia (requisitos, diseño, manuales, estándares).
3 Características a Probar
Lista de funcionalidades o módulos a probar.
Ejemplo: Módulo de Login, Registro de Usuarios, Carrito de Compras.
4 Características a NO Probar
Indicar explícitamente lo que queda fuera de este plan y el motivo.
Ejemplo: Pruebas de carga, Módulo de administración.
5 Enfoque de la Prueba
Descripción de la estrategia general de pruebas.
Ejemplo: Se realizarán pruebas funcionales de caja negra para cada historia de usuario, seguidas de pruebas de regresión automatizadas.
6 Criterios de Pasa/Falla
Definir claramente qué significa que una prueba pase o falle.
Ejemplo: Un caso pasa si el resultado actual coincide con el esperado. Falla en cualquier otro caso.
7 Criterios de Suspensión y Reanudación
¿Cuándo se detienen las pruebas? ¿Qué condiciones deben cumplirse para retomarlas?
Ejemplo: Si se encuentra un defecto bloqueante en el módulo de Login, las pruebas se suspenden. Se reanudan cuando dicho defecto sea corregido y verificado en una nueva build.
8 Entregables de la Prueba
Documentos a generar durante y después de la ejecución: Plan de Pruebas, Casos de Prueba, Reportes de Defectos, Reporte Final de Pruebas.
9 Entorno de Pruebas
Describir hardware, software, herramientas y datos necesarios.
Ejemplo: Servidor de pruebas con Windows Server 2019, Base de Datos SQL Server, Navegadores Chrome y Firefox.
10 Cronograma y Recursos
Indicar hitos clave (inicio, ejecución, cierre de pruebas). Roles y responsabilidades (ej. QA Tester, Líder de Pruebas, Desarrollador).
