🔍 ¿Qué son las Pruebas de Seguridad?

🎯

Definición

"Evaluaciones diseñadas para identificar vulnerabilidades y debilidades en el software"

Fuente: Mahfuz, Cap. 6
🔐

Confidencialidad

Protege:

  • • Datos personales
  • • Información financiera
  • • Secretos comerciales
  • • Credenciales de acceso

Amenazas: Espionaje, robo de datos, acceso no autorizado

✅

Integridad

Garantiza:

  • • Datos no modificados
  • • Autenticidad
  • • Consistencia
  • • Trazabilidad

Amenazas: Manipulación, corrupción, falsificación

⚡

Disponibilidad

Asegura:

  • • Acceso cuando se necesita
  • • Tiempo de respuesta
  • • Continuidad del servicio
  • • Recuperación rápida

Amenazas: DoS/DDoS, fallos de sistema, sabotaje

🎯 Objetivo Principal

Garantizar Confidencialidad, Integridad y Disponibilidad (CIA)

Proactivo
Encontrar antes que atacantes
Sistemático
Metodología estructurada
Continuo
Durante todo el ciclo de vida

⚠️ Tipos de Amenazas Comunes

💉

Inyección SQL

CRÍTICO

¿Qué es?

Inserción de código SQL malicioso en campos de entrada para manipular la base de datos

Impacto:

  • • Robo de datos completos
  • • Modificación de registros
  • • Eliminación de tablas
  • • Bypass de autenticación
🕷️

Cross-Site Scripting (XSS)

ALTO

¿Qué es?

Inyección de scripts maliciosos en páginas web vistas por otros usuarios

Impacto:

  • • Robo de cookies/sesiones
  • • Redirección maliciosa
  • • Keylogging
  • • Defacement de sitio
🔑

Robo de Credenciales

ALTO

¿Qué es?

Obtención no autorizada de nombres de usuario, contraseñas y tokens de acceso

Métodos:

  • • Phishing
  • • Brute force
  • • Credential stuffing
  • • Man-in-the-middle
💥

Denegación de Servicio (DoS/DDoS)

MEDIO

¿Qué es?

Ataques diseñados para hacer que un servicio no esté disponible para usuarios legítimos

Tipos:

  • • Volumétricos (bandwidth)
  • • De protocolo (SYN flood)
  • • De aplicación (HTTP flood)
  • • Distribuidos (DDoS)

📋 Guía de Discusión

🚩 Señales de Alerta Comunes

  • • No usa HTTPS (sin candado)
  • • Pide permisos excesivos
  • • Contraseñas débiles aceptadas
  • • Sin autenticación de dos factores
  • • Errores que revelan información técnica

⚠️ Experiencias Problemáticas

  • • Cuentas comprometidas
  • • Spam después de registrarse
  • • Datos personales expuestos
  • • Transacciones no autorizadas
  • • Sesiones que no expiran

✅ Características de Seguridad Esperadas

  • • Conexión segura (HTTPS)
  • • Autenticación robusta
  • • Notificaciones de actividad
  • • Control de privacidad
  • • Logout automático
