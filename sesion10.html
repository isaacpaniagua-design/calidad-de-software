Definiendo y midiendo la calidad del producto, no solo del proceso

Notas para el docente:

SQuaRE

¿Qué es (y qué no es) SQuaRE?

El Plano de la Calidad del Software

🔄 Recordemos

ISO 9001 se enfoca en la calidad del proceso de gestión (la fábrica)

CMMI se enfoca en la madurez del proceso de desarrollo (la línea de ensamblaje)

SQuaRE es diferente. Se enfoca exclusivamente en la calidad del PRODUCTO FINAL (el coche ya construido)

¿Cuál es su propósito?

Proporcionar un marco de trabajo y un vocabulario común para definir, medir y evaluar la calidad de un sistema o producto de software.

Nos ayuda a responder la pregunta: "¿Qué significa que este software específico sea 'de calidad'?"

Sus Dos Modelos Principales:

Modelo de Calidad del Producto

Define las 8 características internas y externas del software en sí. (Nuestro foco hoy)

Modelo de Calidad en Uso

Mide la calidad desde la perspectiva del usuario final en un contexto real (Eficacia, Eficiencia, Satisfacción, Seguridad en el uso y Cobertura de contexto).

Explorando las Características (Parte 1)

Las Características "Visibles" para el Usuario

✅

1. Adecuación Funcional

Pregunta: ¿El software hace lo que se supone que debe hacer?
Ejemplo:

En una app de e-commerce, el botón "Añadir al carrito" efectivamente añade el producto al carrito con el precio y la cantidad correctos.

🛡️

2. Fiabilidad

Pregunta: ¿Funciona de manera consistente y sin fallos durante su operación?
Ejemplo:

Una app de mapas no se cierra inesperadamente en medio de una ruta o pierde la señal del GPS constantemente.

👤

3. Usabilidad

Pregunta: ¿Qué tan fácil, eficiente y agradable es de usar para el usuario final?
Ejemplo:

Una app de banca móvil permite hacer una transferencia en pocos pasos intuitivos, con botones claros y mensajes de ayuda.

⚡

4. Eficiencia de Desempeño

Pregunta: ¿Qué tan rápido responde y cuántos recursos (CPU, memoria, batería) consume?
Ejemplo:

Un editor de video renderiza un clip de 1 minuto en un tiempo razonable sin sobrecalentar el teléfono.

🔍

Explorando las Características (Parte 2)

Las Características "Ocultas" pero Cruciales

🔧

5. Mantenibilidad

Pregunta: ¿Qué tan fácil es para los desarrolladores modificar el sistema, corregir errores o añadir nuevas funciones?
Ejemplo:

El código está bien estructurado, documentado y modularizado, permitiendo a un nuevo programador entenderlo y hacer cambios sin romper otras partes.

📱

6. Portabilidad

Pregunta: ¿Qué tan fácil es transferir el software de un entorno (hardware, sistema operativo) a otro?
Ejemplo:

Una aplicación web funciona correctamente en Chrome, Firefox y Safari sin necesidad de cambios en el código.

🔗

7. Compatibilidad

Pregunta: ¿Puede el software coexistir e intercambiar información con otros sistemas?
Ejemplo:

Una app de calendario puede sincronizar eventos automáticamente con Google Calendar y Outlook.

🔒

8. Seguridad

Pregunta: ¿Qué tan bien protege la información y los datos contra accesos no autorizados?
Ejemplo:

Una app de mensajería utiliza cifrado de extremo a extremo para que solo el emisor y el receptor puedan leer los mensajes.

Asignación 3 - "Calificando la Calidad de una App"

Aplicando SQuaRE al Mundo Real

🎯 Objetivo

Utilizar el modelo ISO/IEC 25010 como una herramienta de análisis para evaluar la calidad de una aplicación real.

📋 Instrucciones

1

Elegir una aplicación

En los mismos equipos del laboratorio anterior, elijan una aplicación móvil o web que todos conozcan bien.

2

Crear una "Ficha de Evaluación de Calidad"

Creen una ficha de evaluación para esa app usando el modelo SQuaRE.

3

Evaluar cada característica

Para cada una de las 8 características de calidad del producto:

  • • Denle una calificación (ej. de 1 a 5 estrellas)
  • • Justifiquen la calificación con al menos un ejemplo concreto basado en su experiencia como usuarios (o como desarrolladores imaginarios para las características ocultas)

💡 Ejemplo de Justificación

🎵

Para "Fiabilidad" en Spotify:

Calificación:
⭐⭐⭐⭐ ⭐ (4/5)
Justificación:

"La reproducción de música es muy estable, incluso con mala conexión. Sin embargo, a veces la app tarda en cargar las portadas de los álbumes o se queda en 'conectando' al iniciar, por eso no tiene 5 estrellas."

📄 Entregable

Un documento o presentación simple con la ficha de evaluación completa.
