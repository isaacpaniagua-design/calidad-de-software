El Gran Salto: De Nivel 2 a Nivel 3

De "Mi Proyecto" a "Nuestra Organización"

Visualización del Cambio

❌ Equipos Aislados (Nivel 2)

Varios equipos de proyecto trabajando en silos aislados, cada uno con sus propias herramientas y métodos.

✅ Herramientas Compartidas (Nivel 3)

Todos los equipos comparten un conjunto común de herramientas y planos organizacionales.

En Nivel 2, aprendimos a gestionar proyectos individuales. Cada proyecto tenía su plan, sus requisitos y su control. Sin embargo, el "cómo" se hacían las cosas podía variar mucho de un proyecto a otro.

El Nivel 3 representa un cambio de mentalidad fundamental:

Nivel 2 (Gestionado)

🎯 Foco: El PROYECTO

Cada proyecto es una entidad independiente.

⚙️ Procesos: Reactivos

Definidos para cada proyecto. El éxito de un proyecto no garantiza el éxito del siguiente.

🧠 Mentalidad:

"Este es el plan para nuestro proyecto."

Nivel 3 (Definido)

🏢 Foco: La ORGANIZACIÓN

Todos los proyectos son parte de un ecosistema organizacional.

🔄 Procesos: Proactivos

Estandarizados para toda la organización. Se crea un "activo organizacional" de procesos.

🧠 Mentalidad:

"Esta es la forma estándar en que nuestra empresa ejecuta todos los proyectos."

🎯 El Objetivo

La consistencia, la predictibilidad y la eficiencia a escala.

El concepto clave en Nivel 3:

Conjunto de Procesos Organizacionales Estándar

Es una colección de procesos bien definidos, documentados y aprobados que describen la forma de trabajo de la organización.

¿Qué incluye?

Procedimientos y Metodologías

Listas de verificación para asegurar completitud

Criterios de Entrada y Salida

Para las tareas y fases del proyecto

Adecuación (Tailoring)

No todos los proyectos son iguales. Un proyecto pequeño y ágil no necesita la misma burocracia que un sistema crítico para un avión.

El Nivel 3 exige que la organización provea guías de adecuación para que cada proyecto pueda adaptar el proceso estándar a sus necesidades específicas, sin omitir lo esencial.

💡 El proceso estándar no es una camisa de fuerza, es un punto de partida inteligente.

Desarrollo de Requisitos (RD - Requirements Development)

Más allá de la gestión (Nivel 2):

No solo gestionamos los requisitos, sino que los analizamos, especificamos y elaboramos. Se enfoca en traducir las necesidades del cliente en requisitos técnicos claros y sin ambigüedad.

Solución Técnica (TS - Technical Solution)

Diseñar y construir:

Selecciona un diseño, lo implementa y realiza pruebas unitarias. Aquí es donde los requisitos se convierten en código y componentes funcionales.

Integración del Producto (PI - Product Integration)

Ensamblando el rompecabezas:

Se asegura de que los componentes de software (desarrollados en TS) se integren entre sí y con otros elementos del sistema de manera controlada, creando el producto final.
