🎯 Estrategias de Prueba en Móviles

Cómo probar de forma efectiva en móviles

🚀 "Estrategia antes que herramientas"

El éxito en testing móvil no depende de tener todos los dispositivos, sino de saber qué, cómo y cuándo probar

🎯 Objetivo: Dominar las estrategias efectivas para testing móvil integral

🧪 Tipos de Pruebas en Móviles

📋 Clasificación Específica para Móviles

✅ Funcionales
🎯 Casos Típicos

Login: Autenticación con usuario/contraseña, biométricos
Navegación: Menús, tabs, gestos de swipe
Notificaciones: Push, locales, badges

⚡ No Funcionales
🎯 Casos Típicos

Rendimiento: Tiempo de carga, fluidez de animaciones
Seguridad: Encriptación, permisos, almacenamiento
Compatibilidad: Diferentes OS, versiones, dispositivos

📡 Pruebas de Red
🎯 Casos Típicos

Cambio de red: Wi-Fi ↔ Datos móviles
Velocidades variables: 3G, 4G, 5G, conexión lenta
Pérdida de conexión: Modo offline, reconexión

👆 Pruebas de Usabilidad
🎯 Casos Típicos

Accesibilidad: VoiceOver, TalkBack, contraste
Gestos táctiles: Tap, swipe, pinch, long press
Orientación: Portrait/landscape, rotación

🎯 Consideraciones Especiales

  • Contexto móvil: Uso en movimiento, con una mano, en diferentes ambientes
  • Interrupciones frecuentes: Llamadas, notificaciones, cambio de apps
  • Recursos limitados: Batería, memoria, almacenamiento, procesamiento
  • Variabilidad extrema: Dispositivos, OS, redes, condiciones de uso

🛠️ Métodos de Ejecución

⚙️ Enfoques para Ejecutar Pruebas Móviles

📱 Pruebas Manuales en Dispositivos Reales
✅ Ventajas

• Experiencia real del usuario
• Detección de problemas de hardware
• Validación de gestos y sensores
• Comportamiento real de red

❌ Desventajas

• Costoso (muchos dispositivos)
• Lento para regresiones
• Difícil de escalar
• Mantenimiento de dispositivos

💻 Simuladores/Emuladores
✅ Ventajas

• Rápido y económico
• Fácil configuración
• Múltiples configuraciones
• Ideal para desarrollo

❌ Desventajas

• No replica hardware real
• Rendimiento diferente
• Sin sensores reales
• Limitaciones de red

🤖 Automatización
✅ Herramientas

Appium: Cross-platform, múltiples lenguajes
Espresso: Android nativo, rápido
XCUITest: iOS nativo, integrado con Xcode

🎯 Cuándo Usar

• Pruebas de regresión
• Casos repetitivos
• Validación en múltiples dispositivos
• Integración continua

🎯 Estrategia Híbrida Recomendada

  • Desarrollo temprano: Simuladores para pruebas rápidas
  • Validación crítica: Dispositivos reales para casos importantes
  • Regresión: Automatización para casos repetitivos
  • Release: Combinación de todos los métodos

💬 Caso Práctico: WhatsApp

📱 Escenario de Prueba

Validar el envío de mensajes de texto en diferentes condiciones de red

✅ Prueba Funcional
📋 Pasos de Ejecución
  1. Abrir WhatsApp
  2. Seleccionar un contacto
  3. Escribir mensaje: "Hola, ¿cómo estás?"
  4. Presionar botón enviar
  5. Verificar que aparezca ✓ (enviado)
🎯 Resultado Esperado

Éxito: El mensaje se envía correctamente y aparece en el chat con el indicador de enviado (✓)

⚡ Prueba No Funcional
📋 Pasos de Ejecución
  1. Configurar red a 3G (velocidad limitada)
  2. Abrir WhatsApp
  3. Enviar mensaje de texto
  4. Medir tiempo desde envío hasta ✓
  5. Repetir 10 veces para promedio
🎯 Resultado Esperado

Métrica: El mensaje debe entregarse en menos de 2 segundos en conexión 3G estable

📊 Análisis del Caso

  • Funcional valida QUÉ: El mensaje se envía correctamente
  • No funcional valida CÓMO: Se envía en tiempo aceptable
  • Condiciones reales: 3G simula conexiones lentas comunes
  • Métricas específicas: < 2 segundos es un SLA medible

💡 Lección Clave

"Un mensaje que se envía pero tarda 30 segundos en 3G no cumple las expectativas del usuario, aunque funcione técnicamente"

Cada equipo diseña 2 pruebas para una app móvil conocida:

Resultado esperado: [Usuario autenticado correctamente]

⚡ Prueba No Funcional

Atributo a probar: [Ej: Tiempo de carga de feed]
Condiciones: [Ej: WiFi, 50 fotos en feed]
Métrica esperada: [Ej: Feed carga en < 3 segundos]
