Bug Report

🐛 Instrucciones para el Alumno

Cuándo presentarlo: Semana 12, Sesión 35 (Gestión de Defectos)

Nota: Este formato es un entregable clave para documentar los defectos encontrados durante las pruebas. Debe llenarse con claridad y precisión, con ejemplos de pasos para reproducir el problema.

1
ID del Defecto
Código único que identifique el defecto.
Ejemplo: BUG-001
2
Título (breve y descriptivo)
Título claro y conciso que describa el problema encontrado.
Ejemplo: El sistema permite login con contraseña vacía.
3
Proyecto / Módulo
Especifica el proyecto y módulo donde se encontró el defecto.
Ejemplo: Proyecto X / Autenticación.
4
Build / Versión
Versión del software donde se detectó el defecto.
Ejemplo: v1.0.5
5
Entorno
SO, navegador, versión DB, etc.
Ejemplo: Windows 10, Chrome 120, DB MySQL 8.
6
Severidad
Crítica, Alta, Media, Baja.
Crítica: Sistema no funciona | Alta: Funcionalidad principal afectada | Media: Funcionalidad secundaria | Baja: Cosmético
7
Prioridad
Urgente, Alta, Media, Baja.
Urgente: Resolver inmediatamente | Alta: Próximo release | Media: Planificar corrección | Baja: Cuando sea posible
8
Estado
Nuevo, Asignado, En progreso, Resuelto, Cerrado, Reabierto.
9
Reportado por / Fecha
Nombre y fecha de reporte.
Ejemplo: Juan Pérez - 15/12/2024
10
Asignado a
Nombre del desarrollador responsable.
Ejemplo: María González (Desarrollador Frontend)
11
Descripción detallada
Explicación clara del problema encontrado.
12
Pasos para reproducir
Lista numerada de pasos para reproducir el defecto.
Ejemplo:
1. Ingresar usuario válido
2. Dejar contraseña vacía
3. Hacer clic en Ingresar
13
Resultado actual
Qué está ocurriendo actualmente (comportamiento incorrecto).
Ejemplo: El sistema permite el acceso.
14
Resultado esperado
Qué debería ocurrir (comportamiento correcto esperado).
Ejemplo: El sistema debe mostrar mensaje 'Contraseña requerida'.
15
Archivos adjuntos
Screenshots, logs, videos, etc.
Ejemplo: screenshot_login_error.png, console_log.txt, video_reproduccion.mp4
