Sesión 20 Estrategia Niveles

🔺 Pirámide de Pruebas (Martin Fowler)

🖥️

Pruebas de UI

Menor cantidad, más costosas

Lentas • Frágiles • Costosas de mantener
🔗

Pruebas de Integración

Algunas automatizadas

APIs • Servicios • Bases de datos
⚡

Pruebas Unitarias

Altamente automatizadas

Rápidas • Confiables • Fáciles de mantener • Gran cobertura

💡 Principio Clave

Más pruebas en la base, menos en la cima. Las pruebas unitarias son baratas y rápidas, mientras que las pruebas de UI son costosas y lentas.

✅ Criterios para Seleccionar Qué Automatizar

🔄

Alta Frecuencia de Ejecución

Pruebas que se ejecutan repetidamente en cada build o release

Ejemplos: Smoke tests, regression tests, pruebas de sanidad

🏢

Casos Críticos para el Negocio

Funcionalidades que generan ingresos o son esenciales para la operación

Ejemplos: Proceso de pago, login, funciones core del producto

🎯

Escenarios Repetitivos y Predecibles

Casos de prueba con pasos bien definidos y resultados esperados claros

Ejemplos: Formularios, validaciones, flujos de datos

Fuente: Tian, Cap. 7
